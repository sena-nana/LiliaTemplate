# 桌面应用模板设计边界

## 0. 适用范围

本文定义模板仓库如何遵循 LiliaUI 的视觉语言,用于指导桌面端业务页面和模板内 UI 改动评审。

模板面向工程工具类桌面应用。它不是营销站点、通用聊天应用或装饰性仪表盘。界面应帮助用户理解当前位置、当前状态、可执行操作和待处理决策。

通用 UI、样式令牌、桌面壳、标题栏、侧栏、设置页、菜单、主题、默认资源和页面基础类的源实现位于 `C:\Files\workspace\LiliaUI`,并通过以下包提供:

- `@lilia/ui`: 组件、壳层、主题、CSS 令牌和页面基础样式。
- `@lilia/config`: 可继承的 TypeScript、Vite、VitePress 和应用配置同步能力。
- `@lilia/tools`: 默认资源、模板检查、迁移和周边工具。
- `@lilia/build`: 开发、构建、Tauri 运行和验证流程封装。

模板仓库只消费这些能力,不在本地维护第二套公共 UI 或工具实现。

## 1. 设计原则

- 克制、清晰、可扫描。
- 信息密度适合重复工作,但不制造噪音。
- 默认保持安静,只在用户必须行动时提高可见度。
- 主内容 > 当前状态 > 过程信息 / 辅助操作。

每个主界面都应让用户快速回答:

- 我现在在哪里?
- 当前页面状态是什么?
- 下一步是否需要我操作?

## 2. 布局系统

- 标题栏、左侧栏、主工作区容器和设置页由 `@lilia/ui` 的桌面壳提供。
- 模板业务页面只填充主工作区内容,不重新实现壳层结构。
- 应用导航、footer 状态和设置入口通过 `src/app.config.ts` 配置后交给 `@lilia/ui` 渲染。

避免落地页式构图、hero 区块、超大标题、装饰面板和营销式卡片流。卡片只用于独立信息组、重复项、弹窗和工具容器,不要嵌套卡片。

## 3. 色彩令牌

优先使用 `@lilia/ui/styles.css` 中提供的 CSS 变量。模板内不要新增公共颜色体系;确有应用专属语义时,先确认是否应进入 LiliaUI。

| 令牌 | 用途 |
| --- | --- |
| `--bg` | 主工作区表面 |
| `--bg-elev` | 标题栏、侧栏、卡片、浮层 |
| `--bg-subtle` | 输入框、代码块、chip、非激活列表表面 |
| `--bg-hover` | 行和轻量控件 hover 背景 |
| `--bg-active` | 激活行、选中 tab、按下状态 |
| `--border-soft` | 低强调分隔线和内部边界 |
| `--border` | 标准组件边界 |
| `--border-strong` | 浮层、激活输入、强交互边界 |
| `--text` | 主文本 |
| `--text-muted` | 次要标签、元信息、非激活控件 |
| `--text-faint` | 占位、路径、弱提示 |
| `--accent` | 当前项、主操作、焦点 |
| `--accent-strong` | 强主操作、需要比 `--accent` 更高对比的强调 |
| `--accent-soft` | 选中底色、轻量主色 hover、连接正常底色 |
| `--accent-text` | 主色按钮上的文字 |
| `--ok` / `--warn` / `--err` | 成功、警告、错误状态 |
| `--ok-soft` / `--warn-soft` / `--err-soft` | 状态底色、危险 hover 或确认态底色 |

soft 令牌只用于状态底色、选中底色和危险 hover,不要扩大成页面背景、装饰块或大面积铺色。

## 4. 样式组织

- 公共主题令牌、reset、字体、滚动条、selection、基础表单控件、壳层样式和页面基础类全部归属 `@lilia/ui`。
- 模板业务页面优先复用 `.page-header`、`.card`、`.kv` 等 LiliaUI 页面基础类。
- 应用专属样式只能放在业务组件的 scoped style 中,避免向模板新增全局 CSS。
- 跨页面复用、壳层专属或组件体系级样式应迁入 LiliaUI,再通过依赖更新回到模板。

## 5. 组件语言

- 熟悉工具优先使用图标按钮:折叠、搜索、设置、扩展、窗口控制。
- 有可用图标时优先使用 LiliaUI 已接入的 `@lucide/vue` 图标约定。
- 需要明确语义的命令使用文字按钮。
- 行需要支持 hover、active、muted 和 disabled 状态。
- 文案短、直、可操作;按钮写动作,状态写事实,提示写影响。
- hover、active 反馈优先改变背景和文字颜色,不要依靠放大、位移、强阴影或高饱和装饰色。

## 6. 桌面壳

桌面壳由 `@lilia/ui` 维护。模板内不得复制标题栏、侧栏、设置页或 resize 行为;需要调整这些规则时,在 LiliaUI 中修改并验证。

模板只负责:

- 通过 `app.config.json` 提供应用名称、产品标题、版本和标识。
- 通过 `src/app.config.ts` 提供导航项、footer 状态和设置文案。
- 通过 `src/routes.ts` 提供业务页面路由。

## 7. 侧边栏

侧边栏交互和尺寸由 LiliaUI 控制。模板配置侧边栏时遵循以下约束:

- 只配置真实可达的导航项和操作,禁止显示未接入功能。
- disabled 项只用于明确的不可用状态,不要用作路线图占位。
- footer 状态必须表达真实应用状态,不要作为装饰色使用。
- 文案保持短、直、可扫描。

## 8. 页面、卡片和浮层

- 页面标题使用 18px/600,说明文字使用 13px 和 `--text-muted`;页面头部只做定位和状态说明,不要变成 hero。
- 卡片圆角 8px,内边距 14px 16px,边框使用 `--border`,只承载独立信息组、重复项、弹窗或工具容器。
- 卡片标题使用 13px/600、`--text-muted`、大写和轻微字距,表达分组而不是制造大标题。
- key-value 列表左右分布,值允许 `overflow-wrap: anywhere`,避免路径、版本号或长标识撑破布局。
- 菜单、下拉和上下文菜单使用 `--bg-elev`、边框令牌、4px 内边距和克制阴影;菜单项高度保持 28px 左右。
- 弹窗宽度使用视口约束,确认类危险操作只在危险项 hover、pending 或确认态使用错误色。
- 通用浮层、下拉、上下文菜单和确认弹窗来自 `@lilia/ui`;模板业务页面只传入真实菜单项和处理函数。

## 9. 动效和状态

- 常规 hover、active、边框和文字颜色过渡约 0.12s,保持及时反馈。
- 侧栏折叠使用 `cubic-bezier(0.2, 0.8, 0.2, 1)`,同步处理 opacity、transform 和 visibility。
- 折叠状态下隐藏拖拽线并禁用 pointer events;resize 状态下移除不必要 transition。
- 弹窗进入退出可以使用轻微透明度和位移,但必须支持 `prefers-reduced-motion: reduce`。
- loading、empty、disabled 状态不能改变控件固定尺寸或造成布局跳动。

## 10. 设置页

设置页由 `@lilia/ui` 提供默认外观、主题和关于页。模板只通过配置提供应用文案和元信息;新增通用设置分类应先进入 LiliaUI,应用专属设置才放在模板业务边界内。

## 11. 评审清单

- 界面仍然像克制的工程工具。
- 主内容强于状态和辅助控件。
- 新颜色来自 `@lilia/ui` 令牌。
- 公共 UI、公共样式和壳层行为没有在模板内重复实现。
- 应用专属样式限制在业务组件 scoped style 内。
- 深浅主题都可读。
- 控件在 hover、active、loading、empty 状态下尺寸稳定。
- 标题栏、侧栏和页面基础组件保持 LiliaUI 既有紧凑尺寸。
- 动效不抢注意力,并尊重 reduced motion。
- 浮层、菜单和危险状态使用克制边框、阴影和状态色。
- 侧边栏配置只包含真实可用的导航和操作。
- 设置页、菜单、主题和默认资源遵循 LiliaUI 源实现。
