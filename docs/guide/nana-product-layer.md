# Nana 产品能力

Nana 预设使用模板自有的产品 adapter 连接真实业务边界，公共组件与交互模式仍由 `@lilia/nana-ui` 提供。

## 页面与能力

- 首页：任务入口、最近项目加载、撤销反馈与失败恢复。
- 编辑：选择后才显示上下文面板，自动保存明确显示保存中、失败和已保存状态。
- 设置：常用项优先，高级项默认折叠；密度切换直接更新 Nana policy。
- 首次设置：五步进度可跳过、稍后继续、失败重试并从本地状态恢复。

模板将产品逻辑拆到 `src/capabilities`，包括产品 adapter、运行状态、自动保存、恢复和撤销。能力通过应用注入安装，页面不直接依赖 mock 实现。

## 接入真实产品

替换 `createMockProductAdapter` 的创建位置，保持 adapter 契约不变即可。失败应转成用户可理解的影响与恢复动作；技术详情可保留在模型或开发诊断中，但不显示在产品界面。

用于自动化验证失败路径时可设置：

```bash
VITE_TEMPLATE_MOCK_FAILURE=1 yarn dev
```

该变量只控制模板 mock 的首次连接失败，不改变生产产品 adapter 的设计。
